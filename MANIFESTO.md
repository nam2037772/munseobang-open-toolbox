# 문서방 선언문

## 문서 우선 원칙

문서방 프로젝트에서는 코드를 먼저 이해하지 않는다. 먼저 `MANIFESTO.md`를 읽고, 그 다음 `PROJECT_LOG.md`를 읽은 뒤 코드를 분석한다.

철학보다 코드를 우선해서는 안 된다. 프로젝트 문서는 프로젝트의 기억이다. 새로운 AI나 개발자가 참여하더라도 이 문서만 읽으면 같은 철학으로 이어서 개발할 수 있어야 한다.

## 정체성

문서방은 건설인을 위한 오픈 툴박스다.

문서방은 커뮤니티가 아니라 신뢰 기반 레퍼런스 플랫폼이다. 자료창고도 아니고, AI 검색 사이트도 아니다. 문서방은 건설인이 검증된 답과 도구를 바로 꺼내 쓰고, 결과물을 자기 PC에 저장하는 실무 도구다.

## 철학

우리는 정보를 모으지 않는다. 우리는 답을 정리한다.

정보의 양보다 정확도와 신뢰를 우선한다. 사용자가 수많은 검색 결과를 비교하지 않도록, 미리 검토한 답안 카드와 도구 카드로 안내한다.

문서방은 많은 글을 보여주는 곳이 아니다. 건설 실무자가 지금 판단하거나 작성해야 하는 일을 기준과 도구로 바로 처리하게 돕는 곳이다.

## 우선하는 근거

- 정부 공식자료
- 법령
- KS
- KCS
- KDS
- 표준시장단가
- 공신력 있는 기관의 고시와 지침
- 현장에서 반복 검증된 실무 양식

공식자료를 우선하고, 실무 자료는 출처와 검토 필요성을 함께 표시한다.

## 하지 않을 것

- 로그인
- 회원가입
- 게시판
- 커뮤니티 기능
- 사용자 문서 서버 저장
- 사용자 파일 업로드 후 서버 보관
- 검증되지 않은 정보를 정답처럼 제공
- AI가 즉석에서 검색 결과를 만들어내는 검색 서비스

## 검색 정책

문서방의 검색은 AI 검색이 아니다. 검색은 검증된 답안 검색이다.

검색은 미리 정리한 답안 카드와 도구 카드를 빠르게 찾기 위한 프론트엔드 필터링이다. 검색 결과는 많을수록 좋은 것이 아니다. 정확할수록 좋다.

검색 결과 카드는 다음 요소를 갖춘다.

- 제목
- 분류
- 한 줄 답
- 설명
- 신뢰 라벨
- 제공 기능
- 근거 또는 출처
- 열기 버튼

검색 결과가 관련 글 목록처럼 보이면 안 된다. 정리된 답으로 이동하는 카드처럼 보여야 한다.

## 저장 정책

문서방은 사용자 문서를 서버에 저장하지 않는다. 사용자가 만드는 결과물은 사용자의 PC에 저장한다.

공사일보, 사진대지, 계산기 같은 도구는 브라우저에서 처리하고 결과물을 사용자의 PC에 저장하는 방식을 우선한다. 향후 기능을 확장하더라도 서버 보관을 기본 전제로 삼지 않는다.
