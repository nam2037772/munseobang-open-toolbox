# 문서방

**건설인 오픈 툴박스**

문서방은 커뮤니티가 아닙니다. 자료창고도 아니고, AI 검색 사이트도 아닙니다.

문서방은 건설인이 검증된 답과 도구를 바로 꺼내 쓰고, 결과물을 자기 PC에 저장하는 신뢰 기반 오픈 툴박스입니다.

> 우리는 정보를 모으지 않는다. 우리는 답을 정리한다.

## 핵심 원칙

1. 회원가입을 요구하지 않습니다.
2. 사용자 문서와 현장자료를 서버에 저장하지 않습니다.
3. 정부 공식자료, 법령, KS, KCS, KDS, 표준시장단가 등 공신력 있는 자료를 우선합니다.
4. 사용자는 앱 이름보다 업무 카테고리로 접근합니다.
5. 커뮤니티와 토론은 기존 카페가 합니다. 문서방은 기준과 도구를 제공합니다.
6. AI는 전면 기능이 아니라, 향후 필요한 곳에서만 보조 도구로 다룹니다.

## 현재 정보구조

문서방 첫 화면은 전체 앱 카드 나열이 아니라 업무 폴더형 구조입니다.

기본 폴더:

- 공사
- 공무
- 품질
- 안전
- 환경
- 감리
- 기준·법령
- 계산기
- CAD 자료실
- 건설실무브리핑

상단에는 고정 기본 즐겨찾기와 localStorage 기반 즐겨찾기 토글이 있습니다. 검색은 전체 도구 데이터를 대상으로 하며, 폴더 안에 숨은 항목도 검색됩니다.

## 기술 스택

- Vite
- React
- TypeScript
- 일반 CSS
- 정적 프론트엔드 데이터: `src/data/apps.ts`

현재 MVP에는 로그인, DB, 서버 저장, 파일 업로드 보관 기능이 없습니다.

## 실행 방법

```bash
npm install
npm run dev
```

프로덕션 빌드:

```bash
npm run build
npm run preview
```

빌드 결과물은 `dist/`에 생성되며 GitHub Pages에 정적 파일로 배포합니다.

## 도구 추가 방법

새 도구는 `src/data/apps.ts`의 `apps` 배열에 추가합니다.

필수 필드:

- `id`
- `title`
- `description`
- `category`
- `tags`
- `status`
- `badge`
- `url` 선택 사항

`category`는 `folders`에 정의된 폴더 ID 중 하나를 사용합니다. URL을 모르는 항목은 삭제하지 말고 `준비중` 또는 `개발중` 상태로 유지합니다.
