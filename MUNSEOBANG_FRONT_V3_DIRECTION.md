# 문서방 프론트 v3 방향 (MUNSEOBANG FRONT V3 DIRECTION)

1. **첫 화면은 건설인이 가장 먼저 여는 작업 시작 화면이다.**
   - 현장 진입 후 3초 내에 본인에게 필요한 서류 작성이나 도구 검색이 가능하도록 극대화된 시인성과 효율성을 제공한다.

2. **검색은 장기적으로 Smart Omnibox 단일 구조로 간다.**
   - 사용자가 AI 질문, 규정 검색, 도구 찾기를 구분해 입력할 필요 없이 단일 옴니박스에서 지능적으로 파싱하여 결과를 매칭한다.

3. **현재의 AI 검색 / Google 검색 2분할 구조는 v2.5 과도기 구조로 본다.**
   - 단기적 인터페이스 확인을 위해 2분할 구조를 유지하되, 내부 비즈니스 로직과 API 결합은 v3의 단일 스마트 검색 형태로의 흡수를 염두에 두고 설계한다.

4. **검색 Provider는 다음을 지원할 수 있어야 한다.**
   - AI (OpenAI, Gemini, Claude, Local LLM 등)
   - Google
   - 문서방 내부 도구 (Ctrl+K 단축키 연동)
   - 국토부 (보도자료, 고시)
   - KCS/KDS (국가건설기준)
   - CSI (건설공사안전관리종합정보망)
   - 건설실무브리핑 (뉴스 피드)

5. **카테고리는 2단계 이하로 유지한다.**
   - 대분류 폴더와 소분류 태스크(Task) 구조만 허용하여 스크롤 및 탭 피로감을 원천 차단한다.

6. **앱이 100개가 되어도 첫 화면은 복잡해지면 안 된다.**
   - 백화점식 나열이 아니라, 자주 쓰는 앱 임시 런처(Quick Launch) 및 태그 필터 기반의 미니멀리즘 대시보드를 유지한다.

7. **AI는 판단자가 아니라 실무 보조자다.**
   - 최종 책임은 건설인 실무자가 법령 및 시방서 원본을 대조하여 검토해야 하므로, AI 답변 결과에는 언제나 명확한 면책 사항과 공식 원본 연결(Link to Source)을 동반한다.

8. **문서방은 서버 저장소가 아니라 Stateless Workbench다.**
   - 유저 데이터의 보안 유지를 위해 서버 저장을 절대 배제하며, 모든 동작 및 임시 버퍼는 사용자의 로컬 런타임 환경에서만 안전하게 휘발된다.
